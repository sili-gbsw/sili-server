### **백엔드 개발 순서**

**Phase 1 — 기반 (Foundation)**

| 순서 | 기능 | 비고 |
| --- | --- | --- |
| 1 | F-10. 설정(config) 관리 | MongoDB config 컬렉션 1건 시드 + 로더 |
| 2 | F-02. 부품 마스터 관리 | 모든 타점 데이터가 part_id로 마스터 조회 |

**Phase 2 — 핵심 데이터 흐름 (Core Pipeline)

| 순서 | 기능 | 비고 |
| --- | --- | --- |
| 3 | F-01. 공정 데이터 수집(Ingestion) | 판정로직 호출. POST /weld-events → MongoDB weld_events 저장 |
| 4 | F-04. 강제 격상 사전 체크 | (판정 로직) 두께 비 / 전극 현상 / 재질 불일치 분기 4가지. |
| 5 | F-05. 판정 엔진(Scoring) | (판정 로직) 점수 산출. 시나리오 단위 테스트로 검증  |

—> 여기까지가 “PLC에서 데이터를 받아 점수를 매기는” 최초 시스템 완성

**Phase 3 — 운영 흐름(Downstream)**

| 순서 | 기능 | 비고 |
| --- | --- | --- |
| 6 | F-06. 재검 큐 관리 | F-05가 빨간불 또는 강제 격상 시 자동으로 reinspection_queue에 적재.  |
| 7 | F-09. 이력·추적성 | 읽기 전용 API (필터·페이지네이션·CSV 내보내기) |
| 8 | F-07. 현장 알림(ws) | F-05,6이 만든 결과를 WebSocket으로 푸시 |

**Phase 4 — 학습 루프 닫기(Closing the Loop)**

3σ: 3시그마 범위를 벗어난 데이터는 **'발생 확률이 0.3%에 불과한 아주 예외적인 값'**

| 순서 | 기능 | 비고 |
| --- | --- | --- |
| 9 | F-03. 정상 범위 자동 학습 | 데이터가 쌓이기 시작한 시점에 평균·3σ 산출 |
| 10 | F-08. 학습 환류(Feedback Loop) | F-06. 재검 결과 입력 → F-03. 정상 범위 갱신 |

**Phase 5 — 확장(v2)**

| 순서 | 기능 | 비고 |
| --- | --- | --- |
| 11 | F-11. 최소 피치 / 최소 Lap 체크 | 우선 config 및 계산식만 자리 잡아두고 활성화는 보류 |
